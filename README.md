# 📊 SQL Customer Spending Analysis

## 📌 Project Overview

This project analyses customer order and spending behaviour using SQL.
The goal is to extract meaningful insights from transactional data, focusing on customer performance, spending patterns, and purchasing trends.

---

## 🎯 Objectives

* Analyse customer spending behaviour
* Identify high-value customers
* Explore order frequency and average transaction value
* Understand regional spending patterns

---

## 🛠️ Tools & Technologies

* SQL

---

## 📂 Dataset

The dataset contains transactional sales data, including:

* Customer ID
* Order ID
* Order Value
* Location (City/Country)
* Transaction details

---

## 🔍 Key Analysis Performed

* Total spending by country
* Customer order frequency and average value
* Top 5 customers by order volume
* Average spending by city
* Identification of high-value transactions
* Highest spending per customer using subqueries

---

## 📈 Key Insights

* A small number of customers contribute significantly to total revenue (high-value customers)
* Spending patterns vary across regions, highlighting differences in customer behaviour
* Certain customers place frequent orders with higher average values

---

## 💡 Conclusion

This analysis demonstrates how SQL can be used to explore transactional data and uncover patterns in customer behaviour.
The insights can support better customer segmentation and data-driven decision-making.

---

## 🚀 Skills Demonstrated

* SQL Joins (INNER JOIN)
* Aggregations (SUM, COUNT, AVG)
* Filtering (WHERE)
* Grouping (GROUP BY)
* Subqueries
* Data analysis for business insights
